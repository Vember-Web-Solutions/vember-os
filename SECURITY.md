# Security Policy: Vember-OS

## Supported Versions
We currently only provide security updates for the **Main MVP Branch**.

| Version | Supported          |
| ------- | ------------------ |
| MVP     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability
**Do not report security vulnerabilities via public GitHub Issues or Discussions.**

If you discover a security hole in the Vember Core, our Resource Group logic, or the Hybrid Auth handshake, please use the **Private Vulnerability Reporting** feature on this repository.

### Our Process
1. Use the "Report a vulnerability" button under the Security tab.
2. Provide a detailed description of the exploit.
3. The Vember-Web-Solutions team will acknowledge your report within 48 hours.
4. We will coordinate a fix before any public disclosure.

Thank you for helping keep the Vember Ecosystem secure. 🔱
